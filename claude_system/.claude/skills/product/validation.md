# ✅ Skill: Validation

> Validates product assumptions before committing development resources.

---

## Trigger

When the user has an idea or assumption that needs validation before implementation.

## Validation Levels

### Level 1: Desk Research (1 hour)
- Search for existing solutions
- Analyze competitor features
- Review relevant forums/communities
- Check market size data

### Level 2: User Interviews (1 day)
- Prepare 5-7 open-ended questions
- Interview 3-5 target users
- Document pain points and workflows
- Identify patterns across interviews

### Level 3: Prototype Testing (1 week)
- Build a minimal clickable prototype
- Test with 5-10 users
- Measure task completion rate
- Collect qualitative feedback

### Level 4: MVP Launch (2-4 weeks)
- Build minimal feature set
- Deploy to limited audience
- Track key metrics (activation, retention, conversion)
- Iterate based on data

## Validation Checklist

- [ ] Problem is confirmed by real users (not assumed)
- [ ] At least 3 people would pay for this solution
- [ ] No existing solution adequately solves this
- [ ] We have the technical capability to build it
- [ ] Time-to-market is acceptable
- [ ] Unit economics make sense

## Output Format

```markdown
## Validation Report

**Hypothesis**: [What we're testing]
**Method**: [Level 1-4 validation approach]
**Evidence**: [Data collected]
**Verdict**: ✅ Validated | ❌ Invalidated | ⚠️ Inconclusive
**Recommendation**: [Proceed / Pivot / Kill]
```
