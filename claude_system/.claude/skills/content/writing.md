# ✍️ Skill: Technical Writing

> Produces clear, structured technical documentation.

---

## Trigger

When the user needs documentation, README files, API docs, or technical guides.

## Document Types

| Type | Purpose | Template |
|------|---------|----------|
| README | Project overview | Title, description, setup, usage |
| API Docs | Endpoint reference | Method, path, params, response |
| ADR | Decision record | Context, decision, consequences |
| Runbook | Operational guide | Steps, troubleshooting, rollback |
| Tutorial | Learning guide | Prerequisites, steps, examples |

## Writing Rules

1. **Lead with the conclusion** — Put the most important info first
2. **Use active voice** — "The function returns..." not "A value is returned by..."
3. **One idea per paragraph** — Short, scannable paragraphs
4. **Use code examples** — Show, don't just tell
5. **Include edge cases** — Document what happens when things go wrong
6. **Version your docs** — Date and version every document

## Quality Checklist

- [ ] Title clearly describes the content
- [ ] Table of contents for docs > 3 sections
- [ ] All code examples are tested and runnable
- [ ] No jargon without explanation
- [ ] Links to related documentation
- [ ] Updated date is current
